## PSR-12 File Format

Follow PSR-12 coding standard for file format, indentation, and line length.

**Why it matters:** Consistent formatting improves readability and maintainability across the codebase.

**PSR-12 requirements:**

- Encoding: **UTF-8** (no BOM)
- Line endings: **LF** (`\n`)
- Indent: **4 spaces** (never tabs)
- Max line length: **120 characters**
- One blank line at end of file
- Opening `{` on the next line for classes and methods
